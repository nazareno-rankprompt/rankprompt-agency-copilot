# Writing content AI actually cites

The goal isn't "good content." It's content AI will **lift and cite** as the source for a question your buyer asks.

## Rule zero: improve before you create

Check the sitemap first (the own-source autopsy in `brain.md` §3.5). Brands usually already have a page on the topic — a "best X" post, a comparison, an alternatives piece — that simply isn't getting cited. If so, **fix that page; don't write a duplicate** (a second near-identical page splits signal and competes with itself). Only create new when there's a genuine gap. When a page exists but isn't cited, the problem is rarely "not enough content" — it's structure, independence, recency, corroboration, or crawlability (see below + `technical-seo.md`). A vendor's own "best builders" post often loses to an editorial one not because it's worse, but because nothing else echoes it — pair the fix with off-page proof (`outreach.md`, `reddit-social.md`).

## Rule one: the format follows what's already cited

Don't pick a format from a rule. Look at your report: for the category you're targeting, **what kind of pages does AI already cite?** Match it.

- Top cited sources are **how-to guides** → write a how-to guide.
- **Location pages** → build location pages.
- **Listicles** ("best X for Y") → write a listicle.
- **YouTube videos / LinkedIn articles** → produce that.

The citations are AI telling you which format it trusts for this category. Give it that format, done better than the current winners. A beautiful listicle is wasted on a category AI answers from how-to guides.

**If you have no citation data yet**, use the question's intent as a starting guess (then let the report override it):

| Intent | Starting format |
|---|---|
| Awareness | Explainer / FAQ |
| Educational | How-to guide |
| Comparison | "A vs. B" article |
| Alternatives | "[Competitor] alternatives" listicle |
| Consideration | Reviews / pricing page |
| Purchase-intent | "Best [category] for [need]" listicle |
| Use-case | "[Category] for [persona]" page |

## Rule two: write it so a machine can quote it

- **Answer first.** Open each section with the direct answer in a sentence or two, then expand. AI extracts the top.
- **One idea per section**, under clean H2/H3 headings phrased like the question.
- **Cover the sub-questions.** Read the actual answer AI gave for this prompt in the report and pull the sub-angles and comparison dimensions it raises; make each a heading or FAQ. A page that answers every sub-question the model breaks the prompt into is far more citable than one that only answers the headline.
- **Tables for comparisons** — AI reads them row by row.
- **Scannable** — short paragraphs, bullets, bolded key phrases.
- **Trust signals** — a named author, "Reviewed by", a visible "Last updated" date. Refresh quarterly; recency is a real signal.
- **Link out** to authoritative sources — it signals the page is well-researched.

## Rule three: don't ship content alone — build consensus

Format alone rarely wins. AI recommends with confidence when the **same answer is echoed across many independent sources** — your page, a Reddit thread, a few third-party listicles, reviews. That corroboration is often called **digital consensus**, and it's the real prize. So pair every content piece with the off-page proof that fits the category (`outreach.md`, `reddit-social.md`). One great page is a data point; the same recommendation in five places is a consensus — and AI repeats a consensus.

## The community case

If the diagnosis is community-driven (AI sources the category from Reddit/forums), content is half the play: find the exact prompts the community is answering and publish answer-first pages that answer *those same questions* better than the thread — while you're also present in the thread.
